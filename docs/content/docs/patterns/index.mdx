# Patterns

Blockstudio can automatically register WordPress block patterns from file templates. Define your patterns as files, and Blockstudio will parse the HTML-like syntax into WordPress blocks and register them via `register_block_pattern()`.

## Overview

Create a `patterns` folder in your theme directory. Each subfolder containing a `pattern.json` and template file (`index.php`, `index.blade.php`, or `index.twig`) will be registered as a block pattern.

```
theme/
└── patterns/
    ├── hero/
    │   ├── pattern.json
    │   └── index.php
    ├── cta/
    │   ├── pattern.json
    │   └── index.blade.php
    └── banner/
        ├── pattern.json
        └── index.twig
```

## pattern.json

The `pattern.json` file defines the pattern metadata:

```json title="pattern.json"
{
  "name": "hero-section",
  "title": "Hero Section",
  "description": "A hero section with heading and CTA",
  "categories": ["featured", "header"],
  "keywords": ["hero", "banner", "cta"]
}
```

### Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `name` | `string` | *required* | Unique identifier for the pattern (prefixed with `blockstudio/`) |
| `title` | `string` | *required* | Display title in the block inserter |
| `description` | `string` | `""` | Description shown in the inserter |
| `categories` | `string[]` | `[]` | Pattern categories (e.g., `featured`, `header`, `footer`) |
| `keywords` | `string[]` | `[]` | Search keywords for the inserter |
| `viewportWidth` | `number` | - | Preview width in pixels |
| `blockTypes` | `string[]` | `[]` | Associated block types |
| `postTypes` | `string[]` | `[]` | Restrict to specific post types |
| `inserter` | `boolean` | `true` | Whether to show in the inserter |

## Template

The template file contains HTML-like syntax that gets parsed into WordPress blocks:

```html title="index.php"
<block name="core/cover" url="https://example.com/hero.jpg">
  <h1>Welcome to Our Site</h1>
  <p>Your journey starts here.</p>
  <block name="core/buttons">
    <block name="core/button" url="/get-started">Get Started</block>
  </block>
</block>
```

This creates a Cover block containing a Heading, Paragraph, and Buttons block with a Button.

## Block Syntax

Use the `<block>` element to include any WordPress block:

```html title="index.php"
<!-- Core blocks -->
<block name="core/heading" level="2">Section Title</block>
<block name="core/paragraph">Paragraph content here.</block>

<!-- Layout blocks -->
<block name="core/columns">
  <block name="core/column">
    <h3>Column 1</h3>
  </block>
  <block name="core/column">
    <h3>Column 2</h3>
  </block>
</block>

<!-- Custom blocks -->
<block name="blockstudio/section">
  <h1>Welcome</h1>
</block>
```

Standard HTML elements are also supported and automatically converted to blocks:

| HTML Element | WordPress Block |
|--------------|-----------------|
| `<p>` | `core/paragraph` |
| `<h1>` - `<h6>` | `core/heading` |
| `<ul>`, `<ol>` | `core/list` |
| `<img>` | `core/image` |
| `<div>`, `<section>` | `core/group` |
| `<blockquote>` | `core/quote` |
| `<hr>` | `core/separator` |
| `<details>` | `core/details` |
| `<table>` | `core/table` |

## Complete Example

Here's a complete pattern example with a hero section:

**pattern.json:**
```json
{
  "name": "hero-with-cta",
  "title": "Hero with CTA",
  "description": "A full-width hero section with heading, description, and call-to-action buttons",
  "categories": ["featured", "header"],
  "keywords": ["hero", "banner", "cta", "landing"],
  "viewportWidth": 1200
}
```

**index.php:**
```html
<block name="core/cover" url="https://example.com/hero-bg.jpg" dimRatio="60">
  <h1>Build Something Amazing</h1>
  <p>Create beautiful websites with the power of blocks.</p>
  <block name="core/buttons">
    <block name="core/button" url="/get-started">Get Started</block>
    <block name="core/button" url="/learn-more" className="is-style-outline">Learn More</block>
  </block>
</block>
```

## Custom Paths

By default, Blockstudio scans `get_template_directory() . '/patterns'`. You can add additional paths using the filter:

```php
add_filter( 'blockstudio/patterns/paths', function( $paths ) {
    // Add a custom path
    $paths[] = get_stylesheet_directory() . '/custom-patterns';

    // Add a plugin path
    $paths[] = MY_PLUGIN_PATH . '/patterns';

    return $paths;
} );
```

## PHP API

```php
// Get all registered patterns
$patterns = Blockstudio\Patterns::patterns();

// Get a specific pattern
$pattern = Blockstudio\Patterns::get_pattern('hero-section');

// Get registered paths
$paths = Blockstudio\Patterns::get_registered_paths();
```

## Twig Support

If [Timber](https://timber.github.io/docs/) is installed, you can use Twig templates:

**index.twig:**
```twig
<block name="core/cover" url="https://example.com/hero.jpg">
  <h1>{{ "Welcome"|upper }}</h1>
  <p>Dynamic content powered by Twig.</p>
</block>
```

Note: Since patterns are registered at initialization time, Twig templates are compiled without context variables. They're useful for Twig's built-in functions and filters rather than dynamic data.

## Blade Support

If [jenssegers/blade](https://github.com/jenssegers/blade) is installed, you can use Blade templates:

**index.blade.php:**
```blade
<block name="core/cover" url="https://example.com/hero.jpg">
  <h1>{{ strtoupper("Welcome") }}</h1>
  <p>Dynamic content powered by Blade.</p>
</block>
```

Note: Like Twig, Blade templates for patterns are compiled without context variables at initialization time. They're useful for Blade's built-in directives and PHP functions rather than dynamic data.

## Differences from Pages

| Feature | Pages | Patterns |
|---------|-------|----------|
| Storage | Creates WordPress posts | Registered in memory |
| Config file | `page.json` | `pattern.json` |
| Sync | Auto-syncs to database | No sync needed |
| Use case | Predefined page content | Reusable block templates |
| Lock support | Yes (`templateLock`) | No |

Patterns are ideal for reusable block templates that users can insert anywhere, while Pages are for creating specific predefined pages in your site.
