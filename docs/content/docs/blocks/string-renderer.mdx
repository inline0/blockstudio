# String Renderer

The string renderer lets you embed Blockstudio blocks anywhere in your HTML
output using `<bs:>` tags. Blockstudio hooks into the output buffer and replaces
each tag with the full block output.

## Activation

Enable the string renderer in your `blockstudio.json`:

```json title="blockstudio.json"
{
  "stringRenderer": {
    "enabled": true
  }
}
```

Or via the PHP filter:

```php
add_filter('blockstudio/settings/string_renderer/enabled', '__return_true');
```

## Syntax

### Self-closing tags

For blocks without inner content:

```html
<bs:hero title="Welcome" background="dark" />
```

### Paired tags

When you need inner content passed to the block:

```html
<bs:section layout="wide">
  <p>Content inside the block.</p>
</bs:section>
```

### Nested tags

Inner blocks render first, then the outer block:

```html
<bs:section layout="wide">
  <bs:hero title="Welcome" />
  <bs:card title="Features" />
</bs:section>
```

### Namespace prefix

By default, the tag matches the first registered block with that slug. Use a
double-dash to specify a namespace:

```html
<bs:acme--hero title="Branded" />
```

This resolves to exactly `acme/hero`.

## Attribute mapping

Element attributes map directly to block attributes:

```html
<bs:cta title="Get started" count="42" active="true" />
```

String values are passed as-is. Numeric and boolean strings are kept as strings
and converted by the block's attribute definitions.

## Where it works

The string renderer processes the full output buffer, so it works everywhere the
tag appears in the final HTML:

- Post content
- Widget areas
- Theme templates
- Headers and footers
- Shortcode output

## Programmatic usage

Apply the string renderer to any string from PHP:

```php
$html = apply_filters('blockstudio/string_renderer/render', $content);
```

## Components

The string renderer pairs well with [Components](/docs/blocks/components).
Components are blocks that only render programmatically and never appear in the
editor inserter. Use `<bs:>` tags to place them anywhere on the page.

```html
<bs:card title="My Card" description="Card content." />
```
