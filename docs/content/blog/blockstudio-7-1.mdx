Blockstudio 7.1 is the first feature release since the v7 rewrite. It introduces
the string renderer, SCSS support for code fields, and lays the groundwork for
more features to come.

## String renderer

The string renderer lets you embed Blockstudio blocks anywhere in your HTML
output using simple tags. Write `<bs:block-name>` anywhere on the page and
Blockstudio renders the full block output in its place.

```html
<bs:marketing-cta title="Get started" variant="primary" />
```

It hooks into the output buffer, so it works everywhere: post content, widget
areas, theme templates, headers, footers. Anywhere the tag appears in the final
HTML, it gets replaced.

Element attributes map directly to block attributes, so passing data works the
same as it does in the editor.

### Syntax

Self-closing tags for simple blocks:

```html
<bs:hero title="Welcome" background="dark" />
```

Paired tags when you need inner content:

```html
<bs:section layout="wide">
  <p>Content inside the block.</p>
</bs:section>
```

Namespace prefix with double-dash for disambiguation:

```html
<bs:acme--hero title="Branded" />
```

Without a namespace, Blockstudio matches the first registered block with that
slug. With a namespace (`acme--hero`), it resolves to exactly `acme/hero`.

### Activation

The string renderer is opt-in. Enable it in your `blockstudio.json`:

```json
{
  "stringRenderer": {
    "enabled": true
  }
}
```

Or via the PHP filter:

```php
add_filter('blockstudio/settings/string_renderer/enabled', '__return_true');
```

### Programmatic rendering

You can also apply the string renderer to any string from PHP:

```php
$html = apply_filters('blockstudio/string_renderer/render', $content);
```

## SCSS code fields

Code fields now support `language: "scss"`. When set, the field value is compiled
through ScssPhp before rendering as CSS, both in the editor and on the frontend.

```json
{
  "type": "code",
  "id": "styles",
  "label": "Custom Styles",
  "language": "scss",
  "asset": true
}
```

Variables, nesting, mixins, and all standard SCSS features work out of the box.
In the editor, compilation happens server-side with debounced requests so you see
results as you type.

```scss
%selector% {
  $primary: #3b82f6;

  h1 {
    color: $primary;
  }

  &:hover {
    background: lighten($primary, 40%);
  }
}
```

No additional settings required. Just set the language to `scss` and it works.

## Server functions

Blocks can now define PHP functions that the frontend can call. Drop a
`functions.php` into any block directory and Blockstudio handles the rest:
REST endpoint routing, authentication, and a tiny JS client.

```php title="functions.php"
return [
    'subscribe' => function (array $params): array {
        $email = sanitize_email($params['email']);
        return ['success' => true];
    },
];
```

```js
const result = await bs.fn('subscribe', { email: 'user@example.com' });
```

This pairs naturally with the WordPress Interactivity API. Your store actions can
call server functions and update reactive state with the response.

Functions are authenticated by default. Add `'public' => true` to allow
unauthenticated access. Cross-block calling works too: any block can call
functions on any other block by passing the block name as the third argument.
